自组NAS

分享下自己最近组装的NAS

May 17, 2026
3 minute read

引言

最近有了点儿闲钱，正好早就想搞台NAS(Network Attached Storage)玩玩了，然后分析了下自己是不是真的需要NAS之后（拒绝冲动消费），最后还是决定买了，于是便有了这篇文章。

配置如下，性能对我来说完全够用了，顺便吐槽下内存和固态也太贵了吧。

核心服务栈：

系统装的飞牛，为了追求稳定所以没去尝试黑裙。

1. 网络底层与安全过滤

Tailscale：整个异地组网的灵魂。有了它，不管是出门在外还是在学校，都能随时通过虚拟局域网 IP 安全直连家里的 NAS。不用折腾危险的公网端口映射，彻底告别公网 IP 焦虑。飞牛系统自带的我觉得局限性太大，所以没有使用。

AdGuard Home：全屋的 DNS 大管家。在本地直接作为第一道过滤网，静默拦截各种设备上的垃圾广告与追踪弹窗，顺便做一下局域网内的 DNS 加速。

v2raya：网络环境的底层支撑，负责解决某些容器在拉取资源或更新时的网络连通性问题，懂的都懂。(后续估计会换成软路由)

2. 自动化下载

虽然不看 4K 原盘，但必要的资源留存还是得有高效的工具来跑：

Aria2-Pro + AriaNg：常驻后台的轻量级全能下载神器。配合 AriaNg 的 Web 前端，不管是 HTTP 链接还是磁力种子，都能直接跑满带宽。日常远程丢链接挂机下载，体验极佳。

Jackett：强大的 P2P 种子索引聚合器。作为自动化下载的后端黑盒，一键打通全网资源的搜索通路。

qBittorrent：老牌 PT/BT 专用下载器，主要用来挂一些大文件和高质量的开源镜像。

openlist：主要用来聚合文件协议方便挂载，网页功能对我来说反而没那么常用。

3. 数据同步与音频服务

Rclone：这台服务器的“超级运油车”。配合自建的各种存储后端，我用它来直接挂载和管理云端账户。通过流式传输（Streaming）在本地、VPS 和云盘之间互通资源，不吃本地存储，也不占多余的临时空间。

Navidrome：我不怎么看大片，但我离不开音乐。这是极其轻量级的自建音乐流媒体服务。内存占用低到可以忽略不计，配合第三方手机客户端，串流自己的音乐库。

结尾
